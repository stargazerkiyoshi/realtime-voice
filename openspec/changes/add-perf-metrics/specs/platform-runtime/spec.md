## 新增需求

### 需求：端到端延迟指标
系统必须采集并输出单次发言的端到端延迟指标。

#### 场景：一次完整发言
- **当** 用户完成发言且系统生成语音回复
- **则** 系统输出包含关键里程碑的延迟指标

### 需求：指标输出格式
系统必须以单行 JSON 日志输出每次发言的延迟指标。

#### 场景：日志输出
- **当** 一次发言结束
- **则** 输出的 JSON 记录包含所有可用的延迟字段

### 需求：指标日志标识
系统必须为每条性能指标日志添加独立标识字段，便于集中检索与筛选。

#### 场景：集中检索
- **当** 输出性能指标日志
- **则** 日志包含可用于检索的固定标识（如 `metric_type` 或 `event`）
