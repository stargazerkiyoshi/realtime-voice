## 背景
当前后端已迁移至 Node.js，缺少配套的前端调试界面。新增前端测试控制台用于快速验证链路与会话行为。

## 目标 / 非目标
- 目标：提供最小可用的测试台，用于连接 WebSocket、启动/结束会话、查看日志。
- 非目标：业务级 UI、复杂音频播放或录音链路（可在后续扩展）。

## 决策
- 使用 Vite + React（TS）作为前端构建框架。
- UI 组件库采用 Ant Design，以快速交付为优先。
- 路由：React Router；全局状态：Zustand。

## 风险 / 取舍
- UI 快速搭建优先，样式一致性与个性化度降低。

## 迁移计划
- Phase 1：搭建前端脚手架并接入 Ant Design。
- Phase 2：实现基础路由与会话控制页面。
- Phase 3：补充日志显示与必要文档。
